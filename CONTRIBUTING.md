## How Can I Contribute?

### üêõ Reporting Bugs

Before creating bug reports, please check the existing issues to avoid duplicates.

When you create a bug report, include as many details as possible:

- **Use a clear and descriptive title**
- **Describe the exact steps to reproduce the problem**
- **Provide specific examples** to demonstrate the steps
- **Describe the behavior you observed** and what behavior you expected
- **Include screenshots or animated GIFs** if possible
- **Include your device/browser information**:
  - Device type (phone, tablet, desktop)
  - Operating system and version
  - Browser and version (for PWA)
  - Android version (for APK)

#### Bug Report Template

```markdown
**Describe the bug**
A clear description of what the bug is.

**To Reproduce**
Steps to reproduce the behavior:
1. Go to '...'
2. Click on '...'
3. Scroll down to '...'
4. See error

**Expected behavior**
What you expected to happen.

**Screenshots**
If applicable, add screenshots.

**Device Information:**
- Device: [e.g. iPhone 12, Samsung Galaxy S21]
- OS: [e.g. iOS 15.1, Android 12]
- Browser: [e.g. Safari, Chrome 96]
- App Version: [e.g. 1.1.0]
- Installation Method: [APK / PWA]
- Mode: [Guest Mode / Logged In]

**Additional context**
Any other context about the problem.
```

### üí° Suggesting Features

Feature suggestions are welcome! Before creating a feature request:

- **Check if the feature already exists**
- **Check if it's already been suggested**
- **Be clear about the problem you're trying to solve**

#### Feature Request Template

```markdown
**Is your feature request related to a problem?**
A clear description of the problem. Ex. I'm always frustrated when [...]

**Describe the solution you'd like**
A clear description of what you want to happen.

**Describe alternatives you've considered**
Any alternative solutions or features you've considered.

**Additional context**
Add any other context or screenshots about the feature request.

**Would this work in Guest Mode?**
Consider if the feature should work without authentication.
```

## üìù Pull Request Process (When Source Code is Available)

Currently, the source code is not publicly available, but if you have access:

1. Fork the repository
2. Create your feature branch (`git checkout -b feature/AmazingFeature`)
3. Commit your changes (`git commit -m 'Add some AmazingFeature'`)
4. Push to the branch (`git push origin feature/AmazingFeature`)
5. Open a Pull Request

### Pull Request Guidelines

- **Follow the existing code style**
- **Write meaningful commit messages**
- **Update documentation** for new features
- **Test on multiple devices/browsers**
- **Ensure TypeScript types are correct**
- **Include screenshots** for UI changes
